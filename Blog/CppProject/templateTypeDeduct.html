Let us focus on understanding three main categories of type deduction. Consider a general template as follows: 

        
                    
        template < typename T>
        void PrintOut(intputType input);

We are interested in how printOut function processes the input depending upon intputType.

input is passed by value

When intputType is simply T, a completely new object that is a copy of input is processed by PrintOut. Since a new object will be created, it makes sense to have the following deduction rules when calling PrintOut:
    If type of exp is reference, it is ignored.
    After denying the reference-ness, if exp is constant it should be ignored, too.
Let's check these in action. 

                
                            
                #include < iostream>
                #include < string>
                
                
                template< typename T>
                void PrintOut(T input)
                {
                    intput=intput+2;
                    std::cout << input<< std::endl;
                }
                
                
                int main()
                {
                    const int x = 5;
                    int y = 5;
                    const int &z = y; // z is a reference to y as a constant integer

                    PrintOut(x); // expect 7 be shown 
                    std::cout << x; // expect 5 be shown  

                    PrintOut(y); // expect 7 to be shown 
                    std::cout << y; // expect 5 to be shown
                    
                    PrintOut(z); // expect 7 to be shown
                    std::cout << z; // expect 5 to be shown
                
                    std::cin.get();
                }
